# 2.1 Objetivo da Avaliação

O objetivo desta avaliação é examinar o ecommerce-workshop sob a perspectiva de qualidade de software aplicada a um cenário de e-commerce instrumentado. Embora o repositório seja didático, ele reproduz uma arquitetura próxima da realidade, com frontend principal, microserviços, banco relacional, observabilidade distribuída e diferentes estados de falha. Isso permite avaliar não apenas a existência das funcionalidades, mas também a capacidade do sistema de se manter observável, estável e recuperável.

Essa avaliação é relevante em contexto acadêmico porque o projeto foi construído justamente para análise de incidentes, comparação entre versões e diagnóstico de gargalos. A conclusão esperada não é apenas verificar se o sistema funciona, mas identificar quais partes estão degradadas, quais riscos operacionais já são visíveis no código e quais características de qualidade mais impactam um e-commerce desse tipo.

Os resultados desta análise serão usados para orientar a etapa de requisitos e, posteriormente, a definição de métricas e critérios de julgamento. Em termos práticos, a avaliação apoia decisões sobre estabilidade, priorização de correções, adequação do ambiente de execução e suficiência da instrumentação para monitoramento em tempo de execução.
